# Customer Analytics Suite — Olist E-Commerce

A complete end-to-end customer analytics pipeline built on the **Brazilian Olist E-Commerce dataset**. This project covers four analytical stages: RFM segmentation, Customer Lifetime Value (historical & predictive), and Churn Prediction.

## Notebooks Overview

### 1. `01_RFM_ML_Customer_Seg.ipynb` — RFM Segmentation
Builds Recency, Frequency, and Monetary (RFM) features from transaction data. Applies KMeans clustering (4 clusters) to segment customers into:
- **Cluster 0** — High value, loyal customers
- **Cluster 1** — Recent but low spenders
- **Cluster 2** — At-risk customers
- **Cluster 3** — Low value / inactive

**Output:** `all_customer_rfm_segmentation.csv`, `top_high_value_customers.csv`

---

### 2. `02_CLV_Ecomm.ipynb` — ML-Based CLV (Random Forest)
Predicts Customer Lifetime Value using a **Random Forest Regressor**. Engineers features from RFM metrics and order history, trains a model, and outputs CLV predictions per customer.

**Output:** CLV predictions saved to the `outputs/` folder

---

### 3. `03_CLV_2Types.ipynb` — Historical + Predictive CLV
Calculates two types of CLV:
- **Historical CLV** — `Monetary × Profit Margin (10%)` using RFM output
- **Predictive CLV (3-month)** — Uses the **BG/NBD** model (purchase frequency) and **Gamma-Gamma** model (average transaction value) from the `lifetimes` library

**Output:** `historical_clv_results.csv`, `predictive_clv_output.csv`

---

### 4. `04_Churn_Prediction.ipynb` — Churn Prediction
Labels customers as churned/active based on purchase inactivity threshold. Trains **XGBoost** and **Random Forest** classifiers. Evaluates with classification report and accuracy score.

**Output:** Churn predictions and model evaluation metrics

## Input Files Required

All notebooks read from the **Olist E-Commerce Public Dataset**. Download it from [Kaggle](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce) and place the CSVs in `data/raw/`.

| File | Used By |
|---|---|
| `olist_customers_dataset.csv` | All notebooks |
| `olist_orders_dataset.csv` | All notebooks |
| `olist_order_items_dataset.csv` | All notebooks |
| `olist_order_payments_dataset.csv` | RFM, Churn |
| `olist_order_reviews_dataset.csv` | Churn |

> **Note:** These raw data files are NOT included in this repository. They are publicly available on Kaggle and must be downloaded separately.

## Setup & Installation

### Prerequisites
- Python 3.8+
- Google Colab (recommended) or local Jupyter environment

### Install Dependencies

```bash
pip install -r requirements.txt
```

Or in Google Colab, the notebooks handle installations automatically (e.g., `lifetimes`, `xgboost`).

### Running Order

The notebooks must be run **in sequence** as later notebooks depend on outputs from earlier ones:

```
01_RFM_ML_Customer_Seg.ipynb   →   generates all_customer_rfm_segmentation.csv
        ↓
02_CLV_Ecomm.ipynb             →   uses RFM data for ML CLV
        ↓
03_CLV_2Types.ipynb            →   uses RFM data for Historical + Predictive CLV
        ↓
04_Churn_Prediction.ipynb      →   uses order + customer data independently
```

### Path Configuration

Each notebook has a `BASE_PATH` variable at the top. Update it to match your local or Drive path:

```python
# Google Colab
BASE_PATH = "/content/drive/My Drive/Datasets/Customer_Seg"

# Local
BASE_PATH = "./data/raw"
```
